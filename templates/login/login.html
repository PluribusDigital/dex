Signing In

The production application will leverage CMS's enterprise login capability. For the demo, you can simply pick a fake user from the right side to login as the user.

Technology

Although the prototype is not functionally complete, it is built on a real production stack. The frontend uses the US Web Design Standards and AngularJS. The backend uses Node/Express, PostgreSQL, and Elasticsearch - all deployed to Amazon Web Services.

API Documentation

The system follows an "API-first" design. The same API we use internally to support the frontend user interface can be used by States to integrate with their own systems. We provide interactive API documentation to make it as easy as possible for internal or external developers.
